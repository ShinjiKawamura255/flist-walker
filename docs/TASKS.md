# Task Navigation

This repository does not maintain a canonical local active-work queue. Active work is defined by the issue, pull request, user request, or agent task that initiated it; this document does not mirror that state.

## Current Context

- Current project posture and maintenance priorities: [CURRENT_STATUS.md](CURRENT_STATUS.md)
- Documentation and change-type routing: [INDEX.md](INDEX.md)
- Validation requirements for the active change: [TESTPLAN.md](TESTPLAN.md)

## Completed Work

Completed roadmap and closure records are indexed in [history/INDEX.md](history/INDEX.md). Historical records provide context and evidence, but they are not current instructions or an active backlog.

## Boundary

- `CURRENT_STATUS.md` owns current project posture.
- The invoking issue, pull request, user request, or agent task owns active work scope.
- `history/INDEX.md` owns navigation to completed maintenance records.
- Release-specific records and evidence are indexed separately in [releases/INDEX.md](releases/INDEX.md).
